The Basics of the Web and HTML


How the Internet Works

The function of the internet is to allow computers to communicate information to one another through it. That is made possible through a server, the HTTP protocol, the internet itself, and a browser. Servers are computers that's job is to host files. While on the other hand, the typical user uses their computer to view files that servers host. The HTTP protocol takes requests from the internet and returns the appropriate files based on the request from the browser. The internet communicates with browers, by taking requests from the browser and then displaying the appropriate content on them. An example of how this process works is by, lets say, you open your web browser and you type in google.com then hit enter. Your browser sends its request to go onto google.com via the internet. The internet communicates this request with the HTTP protocol, which then retrives the appropriate files from the server. The server returns the appropriate files via the HTTP protocol, which then communicates this information through the internet, which then finally displays the content on your web browser. HTML



As the photo above shows, HTML stands for Hypertext Markup Language, although Hypertext is one word rather than two. HTML is a tool that allows programmers to make web pages. HTML is a very popular programming language and for that reason the majority of the content on the web is written in HTML. HTML is made up of text, what we see, and markup what the text looks like.

Tags, Elements, and Attributes

Tags are functions enlcosed in angled brakcets. The text imbeded between tags is called the tag's content. To make a word(s) appear bold, a bold tag < b > is needed before the word or words you wish to make bold. To communicate to your computer that you no longer want the bold function to execute on your text, you can use a closing tag, which looks like this for the bold function < / b >. A closing tag is not necessary in every case. For example in the case that you want all your text in a web page to be bold a closing tag would be of no use. Once the computer reads the opening tag it will apply it to all the text that comes after it. Another instance where a closing tag is not necessary, is for void tags. Void tags do not have content and for that reason they have no closing tags. Like the line break tag < br > which causes the text to appear on the next line rather than continuing to appear on the same line if space permits. Another feature that some tags have are attributes. Attributes are inserted in the opening tag for certian fucnitons. An equal symbol is used to assign a value to the attribute. A value is put inbetween quotation marks. Tags can have multiple attributes along with their respective values. A good example of a tag with an attribute is the anchor tag, < a href = "url" > content < / a >, which when used provides access to a link for the user. Click here for more information on attributes. Why Computers are Stupid



I once heard an insructor tell my class that you can think of a computer as a really fast but dumb friend, which makes sense because the slightest error(s) in code can cause the computer to do something completely different than what was intended. Computers cannot self fix these errors, and for that reason programmers have to give very specific instructions to the computer in order for it to perform a desired task. Nonetheless, if you have the ability to talk to a computer, you become the one in charge rather than vice versa.

Block VS Inline Elements

HTML tags can either be block or inline. Block tags can be thought of as forming an invisible box around the content within it. On the other hand, inline tags stay within the line it's on. A couple block tags are the paragraph and div tags. Some of the inline tags we've seen are the bold, emphasis, anchor, image, span, and break tags.
